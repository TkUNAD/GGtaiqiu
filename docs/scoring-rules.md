# 比赛加分规则

本文档整理天梯排位、休闲对局及日常加分的计算方式。后台可在管理端「天梯规则」修改部分参数（写入 `backend/data/settings.json`）。

---

## 1. 对局类型

| 类型 | 条件 |
|------|------|
| **排位赛** | 已开台、双方未超日/周排位上限、挑战名次合规 |
| **休闲赛** | 未开台排位、超排位上限转休闲、挑战不合规等 |

当前默认配置示例（以 `settings.json` 为准）：每日排位 **3** 场，每周 **15** 场。

---

## 2. 排位赛结算（按积分差，非段位名）

计算使用：**胜者积分 − 败者积分**（与段位展示无关）。

| 分差 | 胜者 | 败者 |
|------|------|------|
| **&lt; 50** | +25～30（随机） | −15～20（随机） |
| **50～99** | +10～15 | −25～30 |
| **≥ 100** | +35～50 | **0**（高打低败者不扣分） |

- **平局**：双方排位分不变。
- **提前结束**（未打满局）：上述分数 **÷2**（向下取整）。
- 实现：`backend/rating.py` → `ranked_point_delta()`；结算：`services.finish_match()`。

### 同段 / 邻段举例

- 同段且积分接近（如 1450 vs 1420，差 30）→ 走「&lt; 50」档。
- 邻段刚跨边界（如 1210 vs 1185，差 25）→ 同上。
- 同段但差 ≥100（如 1580 vs 1410）→ 胜者 +35～50，败者 0。

---

## 3. 休闲赛结算

| | 分数 |
|---|------|
| 胜者 | +5（`daily_bonus_valid_match`，可后台改） |
| 败者 | 0 |

提前结束：胜者加分减半。

---

## 4. 对局内炸清 / 接清

对方确认一次即生效（`match_bonus.py`）：

| 类型 | 加分 | 其他 |
|------|------|------|
| 炸清 | +20 | 申报方胜 1 局 |
| 接清 | +15 | 申报方胜 1 局 |

- 同一场第 2 次起（达 `bonus_review_threshold`）需**后台审核**后才加分。
- 申报、胜/负局等操作共用 **60 秒**冷却。

---

## 5. 其他

| 项目 | 规则 |
|------|------|
| 开台奖励 | 管理后台开台，约 **8 分/小时**（可配置） |
| 对局过短 | &lt; 120 秒无效，不计分 |
| 7 天未对战 | 每天扣 **5** 分 |
| 30 天未登录 | 排行榜隐藏 |
| 初始积分 | **1000** |
| 商城兑换 | 需 ≥ **2000** 分 |
