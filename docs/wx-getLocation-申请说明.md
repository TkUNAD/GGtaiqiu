# wx.getLocation 接口权限申请说明

> 须在 [微信公众平台](https://mp.weixin.qq.com) → **开发管理 → 接口设置** 中提交，无法由代码自动代提交。

## 一、申请前检查（必做）

- [ ] 小程序已上传**体验版**或**开发版**，审核员可打开首页看到定位功能
- [ ] `backend/data/venues.json` 中球房已填写真实 **latitude / longitude**（gcj02）
- [ ] **设置 → 服务内容声明 → 用户隐私保护指引** 已按 `docs/隐私与服务声明配置指南.md` 填写（含昵称、头像、定位）
- [ ] `app.json` 中 `permission.scope.userLocation.desc` 与申请理由一致

## 二、申请接口理由（复制到表单，约 300 字内）

```
本小程序为台球俱乐部线下记分与天梯排位工具，用户须到店使用。

使用场景：用户打开小程序进入首页后，需确认当前所在台球俱乐部。系统调用 wx.getLocation（gcj02）在用户授权后获取当前粗略位置，与平台登记球房坐标比对，在首页顶部展示「当前俱乐部」名称及距离，并加载该俱乐部的桌台状态、本俱乐部 TOP10 排行、扫码开台入口。若用户距球房超过 50 米，页面会提示确认定位或手动切换球房，避免误选异地球房导致无法扫码对战。

数据说明：仅在使用小程序时、经用户授权后获取一次位置用于球房匹配，不后台持续定位、不记录轨迹、不用于广告推送。用户拒绝授权后可手动选择球房，仍可扫码与对战。

请审核，谢谢。
```

## 四、选填项

- **使用场景视频（推荐）**：15～30 秒录屏：打开小程序 → 授权定位 → 首页显示俱乐部 → 点击切换 → 扫码区域
- **官网链接**：无则留空

## 五、提交路径

1. 登录 https://mp.weixin.qq.com  
2. **开发管理** → **开发设置** → **接口设置**  
3. 找到 **地理位置** → **wx.getLocation** → **申请**  
4. 粘贴上文「申请接口理由」，上传截图 → **提交**

## 六、常见驳回原因

| 原因 | 处理 |
|------|------|
| 界面看不出定位用途 | 确保体验版含首页俱乐部栏，截图含授权弹窗 |
| 理由与界面不符 | 理由与 `app.json` desc、隐私指引三者统一 |
| 类目不符 | 小程序类目选体育/生活服务/工具等线下服务相关 |
| 未配置隐私指引 | 在「用户隐私保护指引」中勾选并说明位置信息 |

## 七、与本项目代码对应关系
